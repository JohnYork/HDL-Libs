## 是否在编写 HDL 程序时遇到过这些烦恼？

- 想修改 RAM 的深度，却必须重新运行一次 IP 生成向导？看着进度条转圈，设计思路也被打断……
- 设计已完成大半，突然发现顶层输入信号的位宽需要调整。于是，不得不逐个修改所有相关 IP 的位宽参数，一次又一次打开向导、调整参数、等待生成完成……

遇到这些情况时，你可能会想：**如果能通过修改少量代码或参数，就能自动适配顶层输入的变化，HDL 程序的维护会不会轻松很多？**

正是基于这样的愿景，我设计了 **HDL-Libs**。

---

# 什么是 HDL-Libs？

HDL-Libs 是一套用 **SystemVerilog** 实现的、面向 HDL 数据处理和程序设计的常用基础模块集合。  
它包含了各类基本功能单元，以及用于参数计算的**常数函数**。

- **常数函数**：按功能分类组织成 `package`。你只需在代码文件头部通过 `` `include `` 语句包含对应文件，即可在模块代码中以 `package_name::function` 的形式调用这些函数。
- **基本功能单元**：将对应文件加入工程，确保编译器可以正确识别后，即可直接调用和例化。

此外，HDL-Libs 特别为许多常用基本功能单元的接口信号扩展了数组支持，进一步提升了模块的复用性和灵活性。

## 已包含的模块

- **miscs** – 常用的基础常数函数，如位宽计算、自动分级延迟阶数计算等
- **packconv** – 接口信号数组转换模块（合并数组 ↔ 一维数组、合并数组 ↔ 非合并数组等）
- **rams** – 简单双口 RAM、真双口 RAM，以及它们的接口数组扩展版本
- **shifttaps** – 移阶寄存器（固定阶数、可变阶数，支持接口数组扩展）
- **greycode** – 格雷码编码 / 解码 / 连续性检测
- **clkdx** – 跨时钟域信号传输模块
- **fifos** – FIFO 器件（双时钟异步 FIFO、单时钟同步 FIFO，支持接口数组扩展）
- **mux** – 复选器（按索引选通、按选通信号选通，支持接口数组扩展）
- **partmux_byidx** , **partmux_bycs** - 数据分块多路复用器
- **partadapter** , **partadapter_byidx** - 数据分块合路器
- **bitcnt** - 比特位统计器（统计数值的二进制表示中值为0或者1的比特位数）
- **clshift** - 算术 / 逻辑移位器
- **lmbd/rmbd** - 从高位向低位 / 从低位向高位搜索比特位
- **paral_add_packedarray/paral_add_unpackedarray** - 并行加法器（暂时仅支持整数加法）
- **imultconst/umultconst/ilmultconst/ulmultconst** - 常数乘法器，可以配置为不用消耗硬件乘法器资源
- **edge_detectr** - 上升/下降边沿探测器
- **AVALON_BUS** - 序列同步用AVALON总线系列模块库，包含总线节点、前后节点连接器、总线分支器、总线节点镜像映射器等
  - **seqs_errchk** - 序列异常状态检测器
- **iaddsub** - 整数加/减法器
- **icmpr** - 整数比较器
